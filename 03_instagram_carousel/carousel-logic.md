# Carousel Narrative Logic

Use this structure for future Atlas Repo carousel posts.

## Format

- Language: English by default unless the post targets Russian-speaking founders.
- Reading mode: essay carousel, not infographic.
- Text placement: the main text starts near the top of every slide.
- Visual role: premium atmospheric support, not the main content.
- Each repo slide should answer:
  - What the repository is.
  - What company function it can replace or compress.
  - Why a founder should care.
- Tone: sharp, founder-facing, practical, slightly provocative, but not fake-hype.

## Slide Sequence

1. Hook: "10 open-source repositories that can replace entire departments."
2. Thesis: companies used to grow through hiring; now software absorbs parts of the org chart.
3. Repo 1: n8n replaces operations workflows.
4. Repo 2: OpenHands replaces part of junior engineering work.
5. Repo 3: Browser Use replaces repetitive browser work.
6. Repo 4-5: Dify launches AI products; Mem0 gives AI memory.
7. Repo 6-7: Postiz replaces marketing ops; Cal.com replaces scheduling workflows.
8. Repo 8-9: Docmost replaces internal knowledge bases; Supabase replaces backend boilerplate.
9. Repo 10 + stack: Plane replaces heavy project management; combine the stack.
10. CTA: Atlas Repo finds these projects; visit atlasrepo.com / link in profile.

## Caption Pattern

Open with the leverage thesis, not the list.
Explain that people are not obsolete, but small teams now have stronger tools.
Ask the audience to save the post if they are building with a small team.
End with Atlas Repo CTA.
