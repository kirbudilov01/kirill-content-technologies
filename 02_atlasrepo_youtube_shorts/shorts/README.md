# Shorts Workspace

Здесь лежит рабочий набор для коротких роликов. Актуальная версия для съемки:

- `../ATLAS_REPO_SHORTS_V4_SIMPLE_READABLE.md` — 100 простых сценариев, которые можно читать в камеру.
- `../ATLAS_REPO_SHORTS_V4_PARTNER_SCREENCAST_TASKS.md` — отдельные задачи партнеру: что снять, какие экраны, какие ограничения.
- `../ATLAS_REPO_SHORTS_V4_SIMPLE_READABLE.xlsx` — табличная версия для планирования публикаций.

Правило для Shorts: сначала бытовой хук, потом инструмент. Не начинать ролик со слов Hermes, OpenClaw, MCP, repo или framework, если зритель еще не понимает, зачем ему это смотреть.

Формула:

1. Боль или выгода: "AI пишет быстро, но не понимает проект", "перед подпиской на SaaS проверь open-source альтернативу".
2. Простое объяснение: без лекции и внутренней терминологии.
3. Один визуальный proof: экран, demo, ошибка, repo, workflow.
4. Вердикт: `save / test / skip`.
5. Мягкий переход: AtlasRepo как радар полезных инструментов.
