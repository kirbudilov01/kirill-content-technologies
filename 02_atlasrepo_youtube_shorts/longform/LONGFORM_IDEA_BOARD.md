# Long-Form Idea Board

Это не финальные сценарии. Это доска тем и тезисов, которую нужно наполнять перед написанием сценариев. Формат: сначала фрейм конкурента, потом наша адаптация, потом съемочные тезисы.

## Tier 1: первые кандидаты на съемку

### 1. Hermes vs OpenClaw: где быстрее собрать полезного AI-ассистента?

**Фрейм:** X vs Y battle.

**Почему может набрать:** есть конфликт, свежие agent/tools темы, понятный зрителю вопрос "что выбрать".

**Наша адаптация:** не рассказываем историю Hermes/OpenClaw, а делаем один и тот же task в двух системах.

**Главный тест:** агент должен прочитать repo, понять задачу, предложить action plan и выдать полезный артефакт.

**Тезисы:**

1. AI-агентов стало слишком много, но большинство сложно отличить друг от друга.
2. Выигрывает не самый умный, а тот, кто быстрее доводит до результата.
3. Сравниваем setup, понятность, skills/tools, качество результата.
4. В конце не один победитель, а победитель по типу зрителя: dev, founder, marketer.

**Переход на AtlasRepo:** "такие баттлы можно делать регулярно, если есть радар новых repo".

### 2. Claude agents убьют n8n или это кликбейт?

**Фрейм:** X killed Y, or not?

**Почему может набрать:** claim спорный, n8n понятен no-code/automation аудитории, agents сейчас горячая тема.

**Наша адаптация:** честно собрать один workflow двумя способами.

**Главный тест:** взять входные данные, обработать через AI, записать результат в таблицу/файл, отправить уведомление.

**Тезисы:**

1. Люди любят говорить, что новый AI убил старый инструмент.
2. Но automation ценят не за магию, а за стабильность.
3. Agent может быть гибче, n8n может быть надежнее.
4. Вердикт должен быть практичным: где agent, где n8n, где связка.

**Переход на AtlasRepo:** "мы будем отслеживать open-source automation и agent tools как отдельную категорию".

### 3. Я проверил 100 AI repo, вот 10, которые реально стоит открыть

**Фрейм:** I tested a lot so you don't have to.

**Почему может набрать:** экономия времени, широкий вход, подходит dev/marketer/founder аудитории.

**Наша адаптация:** AtlasRepo выступает как источник отбора.

**Главный тест:** у каждого repo показать одну пользу и один подвох.

**Тезисы:**

1. Новых AI repo слишком много.
2. Stars не равны пользе.
3. Нужны критерии: понятность, demo, свежесть, use-case, install pain.
4. Топ-10 не как каталог, а как "что я бы реально оставил".

**Переход на AtlasRepo:** главный use-case сайта: экономить время на поиске.

### 4. Cursor + MCP + rules: как не превратить AI coding в хаос

**Фрейм:** Full guide without boring tutorial.

**Почему может набрать:** Cursor/MCP массовые темы, но многие теряются в настройке.

**Наша адаптация:** не "гайд по кнопкам", а workflow для результата.

**Главный тест:** одна маленькая фича в repo: до rules/MCP и после.

**Тезисы:**

1. AI coding без контекста быстро ломает проект.
2. Rules дают стиль и ограничения.
3. MCP дает AI действия.
4. Тесты и review возвращают контроль человеку.

**Переход на AtlasRepo:** "там будем собирать MCP, rules, starters и инструменты для такого workflow".

### 5. Stop paying for AI SaaS: open-source alternatives that actually work

**Фрейм:** Open-source alternative to paid SaaS.

**Почему может набрать:** деньги, SaaS-fatigue, open-source curiosity.

**Наша адаптация:** проверяем не "бесплатно ли", а "можно ли реально использовать".

**Главный тест:** 5-10 категорий: automation, docs, research, UI, AI chat, agents.

**Тезисы:**

1. Подписки расползаются незаметно.
2. Open-source не всегда дешевле, если установка съедает день.
3. Ищем баланс: цена, контроль, setup, качество.
4. Показываем, где open-source реально выигрывает.

**Переход на AtlasRepo:** "AtlasRepo как база open-source альтернатив".
