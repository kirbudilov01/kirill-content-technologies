# Long-Form Workspace

Здесь не начинаем со сценария. Для длинных видео сначала собираем эффективную идею и структуру удержания, а уже потом пишем текст на 15-20 минут.

Актуальные старые материалы:

- `../atlas_repo_longform_30_from_actual_top_patterns.md` — 30 тем по реальным топ-паттернам из YouTube research.
- `../ATLAS_REPO_LONGFORM_V3_FULL_15_20_MIN_SCRIPTS.md` — черновые длинные сценарии на 1300+ слов, пока использовать как сырье, не как финал.
- `../top_longform_references_from_workbook.csv` — таблица референсов из исходного ресерча.
- `../reference_analysis_summary.json` — краткая структура разобранных YouTube-референсов.

Рабочий процесс:

1. Сначала выбираем конкурентный фрейм: Zero to agent, I tested 500 tools, X killed Y, full guide, build & sell, illegal open-source tools.
2. Потом формулируем нашу адаптацию под AtlasRepo: какой инструмент, какой тест, какая польза, какой переход на сайт.
3. Потом делаем тезисы выпуска: обещание, конфликт, 3 акта, демонстрации, финальный вывод.
4. Только после этого пишем полный сценарий.

Цель long-form: не "обзор репозитория", а трафиковый формат, который уже доказал спрос, но адаптирован под AtlasRepo, open-source инструменты, AI agents, vibe coding и продуктовую пользу.
