# Long-form topic review

Здесь фиксируем редакторские решения по темам. Работаем только с длинными видео: тема, вердикт, новая формулировка, что нужно партнеру найти/снять/проверить.

## Раунд 1: From Zero To Result

### 1. Было: "С нуля до первого AI-агента за 25 минут: я не писал код руками"

**Вердикт:** не берем в таком виде.

**Почему:** слишком абстрактно и сложно с первой секунды. "AI-агент" звучит широко, туманно и не дает обычному зрителю понятного результата. Нужен не агент ради агента, а понятная штука, которую все хотят: деньги, контент, лиды, SaaS, автоматизация рутины, экономия подписок.

**Что делаем вместо:** заменить на тему с конкретным результатом.

**Варианты переформулировки:**

1. **Я собрал AI-систему, которая сама ищет идеи для контента и превращает их в план**
   - Массовый угол: контент, маркетинг, YouTube, рост.
   - AtlasRepo: сервис как источник свежих repo/tools, из которых рождаются темы.

2. **Я собрал штуку, которая каждый день находит новые AI-инструменты вместо меня**
   - Массовый угол: все устали от потока AI-tools.
   - AtlasRepo: фактически объясняем главный use-case сайта.

3. **Я собрал AI-помощника для запуска micro-SaaS: он ищет идею, repo и план сборки**
   - Массовый угол: продукт, деньги, MVP.
   - AtlasRepo: каталог как основа для поиска repo и стеков.

**Лучший кандидат на замену:** `Я собрал штуку, которая каждый день находит новые AI-инструменты вместо меня`

**Задача партнеру:** найти 5-10 свежих AI/open-source инструментов из AtlasRepo или GitHub, снять процесс отбора: лента/поиск, карточка инструмента, GitHub README, quick demo, таблица "save/test/skip". Нужен визуальный proof, что это не абстрактный агент, а понятный радар инструментов.

### 2. Было: "Я собрал micro-SaaS за вечер только из open-source repo"

**Вердикт:** берем.

**Почему:** сильный массовый фрейм. Тут есть продуктовая мечта, деньги, скорость, open-source, понятная интрига "реально ли собрать за вечер". Это намного понятнее, чем абстрактный AI-agent.

**Что надо докрутить:** выбрать нишу SaaS и набор repo. Без ниши тема будет звучать вкусно, но расплывчато.

**Возможные ниши:**

1. **AI content calendar для YouTube/Shorts**
   - Почему: прямо связано с нашей задачей и AtlasRepo.
   - Что делает SaaS: берет тему, repo/tool, референс, генерирует контент-план и задачи на скринкасты.

2. **AI tool radar для маркетологов и фаундеров**
   - Почему: почти прямое позиционирование AtlasRepo.
   - Что делает SaaS: показывает новые инструменты, сортирует по use-case, сохраняет подборки.

3. **Open-source alternatives finder**
   - Почему: массовая боль подписок.
   - Что делает SaaS: "введи платный SaaS, получи open-source альтернативы".

**Лучший кандидат:** `Open-source alternatives finder` или `AI content calendar`. Первый шире для аудитории, второй органичнее для AtlasRepo.

**Задача партнеру:** подумать и предложить 2-3 SaaS-ниши, под каждую найти repo-кирпичи: frontend starter, auth, database, payments/mock payments, AI/research component, UI kit, deploy. Нужны скринкасты: выбор repo, GitHub README, установка/демо, итоговый экран продукта, таблица "что взяли из open-source".

### 3. Было: "От идеи до работающего workflow: AI сам собрал мне контент-машину"

**Вердикт:** берем как основу, надо закрутить вокруг AtlasRepo.

**Почему:** тема понятна широкой аудитории: контент, автоматизация, маркетинг, экономия времени. Она естественно связывает YouTube, Shorts, ресерч, n8n/AI и AtlasRepo.

**Как переформулировать сильнее:**

1. **Я собрал AI-контент-машину: она берет новые GitHub repo и делает темы для Shorts**
   - Более прямой мост к AtlasRepo.

2. **Как я превращаю новые AI-инструменты в 100 идей для Shorts**
   - Массовее и понятнее для creators/marketers.

3. **AI сам собрал мне контент-план на месяц из свежих open-source инструментов**
   - Самая понятная выгода: контент-план на месяц.

**Лучший кандидат:** `AI сам собрал мне контент-план на месяц из свежих open-source инструментов`

**Интеграция AtlasRepo:** AtlasRepo = источник сигналов. Сайт дает свежие repo/tools, дальше workflow превращает их в темы, сценарии, задачи партнеру, таблицу публикаций.

**Задача партнеру:** снять цепочку "источник -> обработка -> результат": AtlasRepo feed, выбранные repo, таблица/Notion/Google Sheet с темами, n8n или другой workflow, финальный контент-план. Нужны кадры до/после: пустая таблица и заполненный план на 30 дней.

## Раунд 2: I Tested 100/500 Tools

### 4. Было: "Я проверил 100 AI-инструментов: 10 реально стоят твоего времени"

**Вердикт:** берем.

**Почему:** сильный конкурентный фрейм. Он продает экономию времени: зрителю не надо самому открывать 100 инструментов, мы уже отфильтровали.

**Что докрутить:** обязательно нужны конкретные инструменты, а не общая болтовня. Идеально брать из AtlasRepo/нашей базы, добивать GitHub/официальными сайтами и показывать короткие proof-экраны.

**Финальная формулировка:** `Я проверил 100 AI-инструментов: 10 реально стоят твоего времени`

**Задача партнеру:** выбрать 100 AI/open-source инструментов из AtlasRepo или нашей базы, отобрать 10 финалистов. Для каждого снять: карточку/название, GitHub или сайт, README/demo, один экран пользы, один экран ограничения. Подготовить таблицу с колонками: `tool`, `category`, `why useful`, `proof screen`, `limitation`, `save/test/skip`.

### 5. Было: "Я открыл 50 новых GitHub repo за день. Большинство мусор, но эти 7 — золото"

**Вердикт:** берем.

**Почему:** хорошая драматургия: много шума, немного золота. Тут есть честность, конфликт и сильный повод для удержания.

**Что докрутить:** не просто "репы", а свежие AI/dev/productivity/open-source repo, которые можно быстро показать в кадре.

**Финальная формулировка:** `Я открыл 50 новых GitHub repo за день. Большинство мусор, но эти 7 — золото`

**Задача партнеру:** собрать 50 свежих GitHub repo из AtlasRepo/базы/GitHub trending по категориям: AI agents, MCP, automation, devtools, marketing/content, open-source SaaS alternatives. Отобрать 7 лучших. Для ролика снять быстрый монтаж "мусор/непонятно/мертвый repo" и отдельные скринкасты 7 победителей: README, stars/last commit, demo, quick use-case.

### 6. Было: "Я протестировал AI tools для маркетинга: какие реально делают работу, а не презентацию"

**Вердикт:** берем идею, но аудиторию можно выбрать гибче.

**Почему:** фрейм рабочий, но "для маркетинга" может быть как хорошим массовым входом, так и слишком узкой вилкой. Нужно выбрать задачу, где будет больше пользы и визуальных proof.

**Варианты задач вместо/внутри маркетинга:**

1. **для контента** — темы, сценарии, ресерч, календарь, нарезки.
2. **для продаж/лидов** — поиск компаний, обогащение данных, outreach drafts.
3. **для фаундеров** — идея, ниша, landing, MVP stack, конкуренты.
4. **для разработчиков** — review, docs, tests, browser QA.

**Лучший кандидат сейчас:** `Я протестировал AI tools для контента: какие реально делают работу, а не презентацию`

**Задача партнеру:** выбрать 10-15 open-source AI tools из AtlasRepo/базы под одну конкретную задачу. Пока приоритет: контент-машина, потому что это связывает наш YouTube, Shorts и AtlasRepo. Снять по каждому: сайт/GitHub, что делает, пример входа, пример результата, ограничение. Если выбираем не контент, партнер должен предложить задачу и список repo до съемки.

## Раунд 3: Stop Paying For X

### 7. Было: "Хватит платить за 5 AI-сервисов: вот open-source замены"

**Вердикт:** берем.

**Почему:** очень понятная массовая боль: подписки. Работает и на dev, и на маркетологов, и на фаундеров.

**Что докрутить:** нужны конкретные пары "платный сервис -> open-source альтернатива", иначе тема будет обещать больше, чем даст.

**Финальная формулировка:** `Хватит платить за 5 AI-сервисов: вот open-source замены`

**Задача партнеру:** выбрать 5 популярных AI/SaaS-сервисов и найти open-source альтернативы через AtlasRepo/базу/GitHub. Снять: pricing paid tool, open-source repo, demo/README, что заменяет, где уступает. Не показывать личные аккаунты и платежные данные. Нужна итоговая таблица: `paid tool`, `open-source alternative`, `what replaces`, `setup difficulty`, `verdict`.

### 8. Было: "Перед тем как купить очередную подписку, проверь эти 10 GitHub repo"

**Вердикт:** берем.

**Почему:** это тот же money-saving фрейм, но мягче и шире. Хорошо подходит под AtlasRepo: "сначала проверь open-source, потом плати".

**Что докрутить:** подобрать 10 repo по категориям, не хаотично.

**Финальная формулировка:** `Перед тем как купить очередную AI-подписку, проверь эти 10 GitHub repo`

**Задача партнеру:** собрать 10 repo в категориях: AI chat/research, automation, content, design/UI, devtools, docs, analytics, agents, browser automation, knowledge base. Для каждого снять GitHub header, README/demo, один use-case и короткий verdict. Источник — AtlasRepo/наша база, если не хватает — GitHub trending/search.

### 9. Было: "Я заменил платный AI stack open-source инструментами. Что сломалось?"

**Вердикт:** переформулировать.

**Почему:** "что сломалось" звучит мутно и не дает конкретного обещания. Нужен понятный стек и понятная задача: что именно заменяем, чем заменяем, какой результат получаем.

**Новая формулировка:** `Я собрал open-source AI stack вместо 5 платных подписок`

**Варианты стека:**

1. **Контент-стек:** research tool + script generator + image/video helper + scheduler/table + analytics.
2. **Dev-стек:** AI editor/assistant + MCP/browser QA + docs generator + test helper + deploy helper.
3. **Founder-стек:** idea research + landing builder + CRM/leads + automation + support bot.

**Лучший кандидат сейчас:** `Я собрал open-source AI stack для контента вместо 5 платных подписок`

**Задача партнеру:** выбрать конкретный стек из 5 функций. Для каждой функции найти paid reference и open-source/tool alternative. Снять экран платного сервиса/pricing, экран альтернативы, короткий demo, итоговую схему стека. Важно: ролик должен показывать не "все бесплатно идеально", а честный вывод: где open-source достаточно, где нужна настройка, где платный SaaS удобнее.

## Раунд 4: X Killed Y

### 10. Было: "Claude Agents убьют n8n или это кликбейт?"

**Вердикт:** фрейм оставить, конкретную тему переписать.

**Почему:** сама механика `X убивает Y` рабочая, но Claude agents vs n8n уже выглядит избито и слишком общо. Нужен популярный сервис или категория, где можно реально показать, что open-source/AI отъедает конкретный use-case.

**Направления на замену:**

1. `Chatwoot против Intercom/Zendesk: можно ли перестать платить за support?`
2. `Bruno против Postman: open-source API-клиент, который живет в Git`
3. `n8n против Zapier/Make: где open-source automation реально выигрывает`
4. `Metabase против дорогих BI-дэшбордов`

**Задача партнеру:** выбрать 1-2 баттла, где есть сильный paid competitor, open-source repo из нашей базы, понятный demo и pricing contrast.

**Вердикт:** все три берем.

**Почему:** tier list понятен зрителю, хорошо удерживает, хорошо режется на Shorts, можно делать на основе AtlasRepo/базы.

**Задача партнеру:** собрать инструменты из AtlasRepo/базы, подготовить proof screens и tier board. Для dev tier list использовать не только AI-chat, а API, local env, observability, browser QA, docs, database, deploy.
