# Kirill Content Technologies

This repository is a sanitized public knowledge base for Kirill's content production system.

It collects practical assets for:

- daily content radar;
- AtlasRepo media production;
- Shorts, Reels, TikTok, and YouTube long-form;
- Instagram carousels;
- X / Threads posting systems;
- article and SEO content generation;
- Russian article factories for non-AtlasRepo niches;
- TrendVi / want2view content-factory and SEO systems;
- avatar / vertical video production;
- partner screencast workflows;
- Codex and agent skills used in the production pipeline;
- future want2view API integration.

## Core Idea

The system is not a static backlog of 100 videos.

The operating model is:

```text
AtlasRepo + want2view + live news + published history
        -> daily radar
        -> main theme of the day
        -> long-form videos
        -> Shorts/Reels/TikToks
        -> carousels
        -> X/Threads posts
        -> articles/forum posts
        -> analytics feedback loop
```

## Public Safety Rules

This repository intentionally excludes:

- real `.env` files;
- API keys and tokens;
- Telegram sessions;
- browser profiles and auth state;
- private runtime config;
- logs with credentials;
- `node_modules`, `venv`, caches, and generated local state.

If you need to run any automation, create fresh local credentials outside the repository.

Treat any credentials that ever lived in local draft docs as compromised and rotate them before real usage.

## Content Cadence

Start mode:

- Instagram: 5-6 carousels/week/account, 10-15 Reels/week/account.
- TikTok: 3 videos/day/account.
- YouTube Shorts: 3 Shorts/day/account.
- YouTube long-form: 2-3 videos/day at the beginning.
- X / Threads: 4-5 posts/day on the personal account.

Scale mode:

- YouTube Shorts: 5-6/day/account after validation.
- Long-form: can scale aggressively if recording/editing/analytics support it.
- Increase volume by repurposing strong topics, not by lowering quality.
